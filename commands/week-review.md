# Weekly Review

Run a weekly review. Get today's date first:

```bash
date
```

Work through each section:

**1. What was accomplished this week** — search knowledge directories for items completed, marked done, or closed in the last 7 days. Also check recent git commits across any project directories for a summary of code work done.

**2. What is planned for next week** — surface any items in knowledge directories scheduled for next week, or flagged as "next" or "upcoming."

**3. What is carrying forward** — items that were due this week but aren't marked done. Flag these clearly — they need either a new date or a decision to drop them.

**4. One thing to improve** — based on what you found, suggest one process or habit improvement for next week. Keep it specific and actionable.

Present each section concisely. The whole review should be readable in 2–3 minutes.

---
*Customise this command for your setup by editing `~/.claude/commands/week-review.md`*
